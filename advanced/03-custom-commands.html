応用編 A3

カスタムスラッシュコマンド

このチャプターのゴール

  • よく使う依頼を / 一発で呼び出せるようにする
  • 引数 $ARGUMENTS でコマンドを汎用化する
  • ! や @ でコマンド実行・ファイル参照を埋め込む

2 種類のカスタムコマンド

  • プロジェクト用: .claude/commands/(名前).md
    そのプロジェクトでだけ使える。Git にコミットしてチームで共有。
  • ユーザー用: ~/.claude/commands/(名前).md
    どのプロジェクトでも使える、自分専用。

ファイル名 (拡張子を除く) がそのまま スラッシュコマンド 名になります。 .claude/commands/test.md なら /test で呼び出せます。

最小構成

ファイルの中身は Markdown で、本文をそのまま Claude に送るプロンプトとして扱います。 

# .claude/commands/test.md

テストを実行して、失敗しているテストがあれば
原因と修正方針を教えてください。

これだけで、入力欄から /test と打つだけで上記の指示が送れるようになります。

引数を受け取る: $ARGUMENTS

コマンドのあとに書いた文字を、ファイル中の $ARGUMENTS に差し込めます。 

.claude/commands/translate.md

以下の内容を $ARGUMENTS に翻訳してください。
原文の意味を変えず、自然な表現を心がけてください。

これで /translate 英語、/translate フランス語 のように使い回せます。 引数を複数欲しいときは $1 $2 も使えます。

フロントマター (オプション)

ファイル先頭に --- で囲んだ フロントマター を書くと、 コマンドの動作を細かく制御できます。 

---
description: PR の説明文を生成する
allowed-tools: Bash, Read
model: claude-opus-4-5
argument-hint: "[ブランチ名]"
---

git diff $ARGUMENTS..HEAD の内容を読んで、
コミットに対応する PR 説明文を Markdown で書いてください。
セクション: 概要 / 変更点 / テスト方法
キー意味
descriptionコマンド一覧 (/help) に出る説明
allowed-toolsこのコマンド実行中に使えるツールを限定
modelこのコマンドだけ別モデルで動かす
argument-hint入力候補に表示されるヒント文字列

シェルコマンドを埋め込む: ! 記法

コマンド本文の中で !`コマンド` と書くと、その時点でコマンドが実行され、出力がプロンプトに埋め込まれます。 

---
description: 現在の git 状態を要約
allowed-tools: Bash
---

以下の git 状態を読んで、変更内容を 3 行で要約してください。

ブランチ: !`git branch --show-current`

変更ファイル:
!`git status -s`

最近のコミット:
!`git log --oneline -5`

Claude が動き始める前に、!`...` の部分が実行された結果に置き換わるので、 最新の状態を確実にコンテキストに含められます。

ファイルを読み込む: @ 記法

@パス と書くと、そのファイルの内容がプロンプトに埋め込まれます。 

以下のスタイルガイドに従ってレビューしてください。

@docs/style-guide.md

レビュー対象: $ARGUMENTS

サブディレクトリで階層化

.claude/commands/git/commit.md のようにサブフォルダに置くと、 /git:commit として呼び出せます。 関連コマンドをグループ化したいときに使います。

実用レシピ

テストグリーン化

.claude/commands/fix-tests.md
---
description: 失敗テストを直してすべてグリーンに
allowed-tools: Bash, Read, Edit
---

以下の手順で進めてください。
1. !`pnpm test` を実行
2. 失敗テストを 1 件ずつ確認
3. テストが正しく、実装が間違っているなら実装を直す
4. テストが間違っているなら、まず確認を求める
5. すべてグリーンになるまで繰り返す

コミットメッセージ生成

.claude/commands/commit.md
---
description: ステージしてある内容からコミットメッセージを作ってコミット
allowed-tools: Bash
---

以下のステージ内容を読んで:

!`git diff --cached`

Conventional Commits 形式でメッセージを作り、
`git commit -m "..."` を実行してください。
本文 (body) は不要、件名 1 行のみ。

新機能の計画

.claude/commands/plan-feature.md
---
description: 新機能の実装計画を作る
argument-hint: "[機能名]"
---

「$ARGUMENTS」という機能を追加するための計画を立ててください。
プランモードに入って:
1. 既存コードのどこに影響するか調査
2. 実装ステップを 3〜7 個に分けて提示
3. テスト戦略を 1 段落で
4. リスクと注意点を箇条書きで
コマンドファイルを編集したら、その場で反映されます。再起動は不要。 入力欄で / を打つと最新のコマンド一覧がポップアップします。

このチャプターのまとめ

  • カスタムコマンドはプロジェクト共有 / ユーザー個人の 2 種類
  • 引数は $ARGUMENTS や $1 $2 で受け取れる
  • フロントマターで description / allowed-tools / model を指定
  • !`コマンド` でシェル出力を、@パス でファイル内容をプロンプトに埋め込める
