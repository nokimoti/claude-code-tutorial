応用編 A8

settings.json リファレンス

このチャプターのゴール

  • 3 階層の設定ファイルの優先関係を整理する
  • 主要キー (permissions / hooks / env / model) を一通り見る
  • 許可ルールの正規表現パターンを書ける

3 つのファイル、3 つの役割

ファイル場所用途Git
settings.json (ユーザー) ~/.claude/settings.json パソコン全体に効く好み —
settings.json (プロジェクト) (プロジェクト)/.claude/settings.json チームで共有するルール コミット
settings.local.json (プロジェクト)/.claude/settings.local.json 自分専用のオーバーライド 除外

上のファイルが下を上書きする形で重ね合わされます。 最も具体的な設定 (settings.local.json) が最終的な動作を決めます。

JSON のおさらい

settings.json は素の JSON です。 コメントは書けない、末尾カンマは禁止、キーは必ず二重引用符 ── これだけ守れば壊れません。

主要キー一覧

permissions — 許可ルール

{
  "permissions": {
    "allow": [
      "Read(*)",
      "Bash(npm test:*)",
      "Bash(git status)",
      "Bash(git diff:*)",
      "WebFetch(domain:docs.claude.com)"
    ],
    "deny": [
      "Bash(rm -rf:*)",
      "Bash(git push --force:*)",
      "Read(.env)",
      "Read(.env.*)"
    ],
    "ask": [
      "Bash(git push:*)"
    ]
  }
}
  • allow — 確認なしで実行
  • deny — 絶対に拒否 (許可モードを Bypass にしても効く)
  • ask — 必ず確認ダイアログを出す

パターンの書き方

  • Read(*) — 全ファイル読み取り
  • Read(src/**) — src 配下のみ
  • Bash(npm test) — 完全一致
  • Bash(npm test:*) — npm test で始まる任意のコマンド (引数あり可)
  • WebFetch(domain:example.com) — example.com に対する fetch のみ
  • mcp__github__* — github MCP サーバーの全ツール

hooks — フック

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write|MultiEdit",
        "hooks": [
          { "type": "command", "command": "pnpm format", "timeout": 30000 }
        ]
      }
    ]
  }
}

詳細は A5: フック実践 参照。timeout の単位はミリ秒です。

env — 環境変数

{
  "env": {
    "NODE_ENV": "development",
    "DEBUG": "claude:*"
  }
}

Claude Code が実行するすべてのコマンドに渡される環境変数。 たとえばテストで使う変数を統一したいときに便利。

model — 既定モデル

{
  "model": "claude-sonnet-4-5"
}

/model での切り替えとは別に、起動時のデフォルトを固定できます。

statusLine — 画面下のステータス表示

{
  "statusLine": {
    "type": "command",
    "command": "echo \"$(git branch --show-current) | $(date +%H:%M)\""
  }
}

アプリ画面下部に表示される情報をカスタマイズできます。 Git ブランチ・時刻・トークン残量など、欲しい情報を 1 行に。

includeCoAuthoredBy — コミットの共著者

{
  "includeCoAuthoredBy": false
}

コミット作成時に Claude を Co-Authored-By に追記するか。デフォルトは true。

cleanupPeriodDays — 履歴の保管期間

{
  "cleanupPeriodDays": 30
}

古い会話履歴を何日まで保持するか。プライバシー要件に応じて短くできます。

完成形のサンプル

.claude/settings.json (プロジェクト用)
{
  "model": "claude-sonnet-4-5",
  "permissions": {
    "allow": [
      "Read(*)",
      "Glob(*)",
      "Grep(*)",
      "Bash(pnpm test:*)",
      "Bash(pnpm typecheck)",
      "Bash(pnpm format)",
      "Bash(git status)",
      "Bash(git diff:*)",
      "Bash(git log:*)",
      "Bash(git add:*)",
      "Bash(git commit:*)"
    ],
    "deny": [
      "Read(.env)",
      "Read(.env.*)",
      "Read(secrets/**)",
      "Bash(rm -rf:*)",
      "Bash(git push --force:*)",
      "Bash(npm install --global:*)"
    ],
    "ask": [
      "Bash(git push:*)",
      "Bash(pnpm install:*)"
    ]
  },
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write|MultiEdit",
        "hooks": [
          { "type": "command", "command": "pnpm format && pnpm typecheck" }
        ]
      }
    ]
  },
  "env": {
    "NODE_ENV": "development"
  }
}

動作確認

  • /permissions — いま有効な許可ルール
  • /config — 設定画面を開いて全項目を GUI で確認
  • /status — 現在のモデル・接続状況・3 階層のマージ結果
新しい settings.json を書いたら、JSON Lint や jq で構文チェックすると安心です。 Claude に「この settings.json を検証して」と頼むのが一番早いかも。

このチャプターのまとめ

  • 3 階層 (ユーザー / プロジェクト / ローカル) で重ね合わせ
  • permissions は allow / deny / ask の 3 種類
  • Bash(コマンド:*) のように接頭辞マッチで広く許可できる
  • hooks / env / model / statusLine などで挙動を細かくカスタムできる
