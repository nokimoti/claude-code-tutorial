応用編 A7

スキルを活用する

このチャプターのゴール

  • スキル が「必要なときだけ読まれる知識パック」だと理解する
  • 標準スキル (PDF / docx / xlsx / pptx) の使いどころを把握する
  • 自作スキルの最小構成を書ける

スキルが解く課題: 「全部覚えさせるとコンテキストが死ぬ」

Claude に PDF 操作や Excel 編集の細かい手順を CLAUDE.md に書くと、毎回読み込まれてトークンを大量に消費します。 でも、使うのは PDF を扱う一部のセッションだけ。

スキル は、この問題を 「必要になったときに必要な分だけ読み込む」 というしくみで解決します。

サブエージェント・カスタムコマンドとの違い

仕組み呼ばれ方主な役割
カスタムコマンド ユーザーが /名前 と打つ 定型プロンプトの再利用
サブエージェント Claude が判断して呼ぶ (or 明示) 独立コンテキストでの専門作業
スキル Claude が判断して読み込む 特定領域の手順書・参考実装

標準で入っているスキル

Anthropic 公式が提供しているスキルがあります。Claude Desktop アプリではすぐ使える状態になっていることが多いです。

  • pdf — PDF の読み取り・結合・分割・OCR
  • docx — Word ファイルの作成・編集 (テンプレート、スタイル、画像)
  • xlsx — Excel ファイルの読み書き、グラフ作成
  • pptx — PowerPoint の生成
  • skill-creator — 新しいスキルを作るための「メタスキル」

たとえば「report.pdf から表だけ抽出して Excel にして」と頼むと、 pdf スキルと xlsx スキルの両方が自動で読み込まれて、適切な手順で進めてくれます。

スキルの構造

スキルは 1 つのフォルダに以下を置いた形をしています。 

.claude/skills/my-skill/
├── SKILL.md          ← メイン (必須)
├── reference.md      ← 詳細リファレンス (必要に応じて)
├── scripts/
│   └── helper.py     ← 補助スクリプト
└── examples/
    └── sample.json   ← サンプルデータ

SKILL.md の最小例

---
name: invoice-parser
description: PDF 請求書から金額・日付・取引先名を抽出する。"請求書" や "invoice" の単語が出てきたら使う。
---

# 請求書パーサー

PDF 請求書を構造化データに変換する手順をまとめたスキル。

## 抽出する項目
- invoice_number (請求書番号)
- date (発行日)
- vendor (取引先)
- amount (合計金額)
- tax (消費税)

## 手順
1. pdf スキルで PDF をテキスト化
2. 以下のパターンで抽出:
   - 請求書番号: "Invoice No.|請求書番号" の右側
   - 日付: ISO 8601 形式に変換
3. 結果を JSON で返す

## サンプル
@examples/sample.json

フロントマターの重要項目

キー意味
name スキルの識別名
description もっとも重要。Claude が「この会話でこのスキルを使うべきか」を判断する根拠
allowed-tools このスキルが使えるツールを限定

description の書き方が運命を分ける

スキルが 必要なときだけ呼ばれる ためには、description が具体的でないといけません。 抽象的だと「いつでも呼ばれる」または「全然呼ばれない」のどちらかになります。

  • ❌ 「データ処理を支援するスキル」
  • ✅ 「CSV や TSV を JSON に変換するスキル。"CSV を JSON に" のような指示で使う」

「いつ使うか」 をはっきり書くのがコツです。「TRIGGER」セクションを設けるパターンも有効。

スキルを作るスキル: skill-creator

標準で入っている skill-creator スキルを使うと、 会話の中で「~~~ するスキルを作りたい」と頼むだけで、適切な構成のスキルを自動で生成してくれます。 

あなた: 法人向け契約書のチェック観点をまとめたスキルを作って。
        固有名詞・金額・期間・自動更新条項を必ず確認するように。

Claude: ● Skill: skill-creator (使用)
        スキル定義の雛形を作りますね。
        以下の構成を提案します:

        .claude/skills/contract-reviewer/
        ├── SKILL.md
        └── checklist.md
        ...

スキルが効いているか確認

  • 使われたスキルは、画面に「📚 Skill: pdf (読み込み)」のように表示される
  • 呼ばれない場合は description の具体性が足りない可能性大
  • 「pdf スキルを使って」と明示的に呼び出すこともできる

共有とパッケージング

スキルは単なるフォルダなので、Git でリポジトリに入れて共有するのが基本です。 組織内で標準スキル集を作って ~/.claude/skills/ にチームで配布する、といった運用もできます。

スキルとサブエージェントは似ていますが、「分離されたコンテキストで動かしたい」ならサブエージェント、 「メイン会話の中で参照したい知識」ならスキル と考えると整理しやすいです。

このチャプターのまとめ

  • スキルは「必要なときだけ読まれる知識パック」
  • 標準で pdf / docx / xlsx / pptx などが入っている
  • SKILL.md + 補助ファイルでフォルダごと共有
  • description で「いつ使うか」を具体的に書くのが鍵
  • skill-creator で対話的にスキルを生成できる
