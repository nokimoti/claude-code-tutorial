応用編 A6

MCP サーバーをつなぐ

このチャプターのゴール

  • MCP サーバーが「Claude に新しい手足を生やす」仕組みであることを理解する
  • 有名な MCP サーバーを 1 つ追加してみる
  • 3 つの接続方式 (stdio / SSE / HTTP) の違いを把握する

MCP は外部システムとの「共通プロトコル」

MCP (Model Context Protocol) は、 Claude のような AI から外部システムを操作するための共通仕様です。 MCP サーバーをつなぐと、Claude が新しい ツール を使えるようになります。

  • GitHub の Issue を読む / 作る
  • Slack にメッセージを投稿
  • Postgres に SQL を投げる
  • ブラウザを操作する
  • Notion / Linear / Jira などのタスクを操作

3 つの接続方式

方式用途例
stdio ローカルでコマンドを起動。最も多い形式 npx @modelcontextprotocol/server-github
SSE (Server-Sent Events) HTTP 経由のストリーミング (古い形式) SaaS 提供のサーバー
HTTP 新しい HTTP/JSON 方式 (推奨) クラウドの公式コネクター

追加方法 1: GUI から (推奨)

  1. アプリ左下の歯車 (⚙) をクリック → 設定画面を開く
  2. 「コネクター」または「MCP サーバー」タブを選ぶ
  3. 「追加」ボタンから、リストから選ぶ or URL を入れる
  4. 必要なら認証 (OAuth / API キー) を入力

公式が用意している有名サーバー (GitHub、Slack、Notion など) は数クリックで導入できます。

追加方法 2: 設定ファイルで

上級者はプロジェクトの .mcp.json に直接書くことで、 Git でチームに共有できます。 

.mcp.json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_TOKEN}"
      }
    },
    "filesystem-docs": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/docs"]
    },
    "linear": {
      "type": "http",
      "url": "https://mcp.linear.app/sse"
    }
  }
}

${VAR} でシェル環境変数を展開できるので、 機密情報をコミットせずに済みます。

有名な MCP サーバー

  • filesystem (@modelcontextprotocol/server-filesystem) — 指定フォルダの読み書き。プロジェクトフォルダ外のファイルに触れさせたいときに便利
  • github — Issue / PR / リポジトリ操作。コードレビューや進捗管理に
  • postgres — DB へのクエリ。スキーマを質問しながら開発できる
  • puppeteer / playwright — ブラウザ操作。E2E テストやスクレイピング
  • slack — メッセージ投稿、チャンネル一覧
  • linear / jira / notion — タスク・ドキュメント連携
  • memory — 永続化メモリ。会話をまたいだ知識の蓄積

使い方

MCP サーバーを追加すると、Claude は何もしなくてもそのサーバーが提供するツールを認識します。 自然言語で頼むだけで、自動で適切なツールを選んで使ってくれます。 

あなた: GitHub で「bug」ラベルが付いた未解決 Issue を 5 件見せて

Claude: ● github__list_issues(state: "open", labels: ["bug"], per_page: 5)
        ⎿  5 件取得

        以下の 5 件です。
        1. #142 ログイン後に画面が真っ白...

権限の管理

MCP サーバーが提供するツールは、通常のツールと同じように 許可モード の対象になります。 settings.json で個別に制御できます。 

{
  "permissions": {
    "allow": [
      "mcp__github__list_issues",
      "mcp__github__get_issue"
    ],
    "deny": [
      "mcp__github__delete_repository"
    ]
  }
}

ツール名は mcp__(サーバー名)__(ツール名) という形式です。

動作確認とデバッグ

  • /mcp — 接続中のサーバー一覧と状態を表示
  • サーバーが提供するツールは入力欄で @ を打って候補から確認できる
  • 失敗したら設定画面のログを見る (タイムアウトや認証エラーが多い)

自分で MCP サーバーを作る

公式 SDK を使えば、TypeScript / Python で 1 ファイルから MCP サーバーを書けます。 社内 API・独自データソース・既存ツールのラップなどに使うと、Claude を真にカスタムなアシスタントにできます。

  • 公式 SDK: github.com/modelcontextprotocol
  • 「Hello, MCP」レベルなら 30 行ほど
第三者製の MCP サーバーは あなたのパソコンでコードを実行 します。 公開元が信頼できるかを必ず確認してから入れてください。 また、必要以上の権限 (DB の DELETE 等) を与えないように deny リストの活用を。

このチャプターのまとめ

  • MCP は Claude に外部接続を生やす共通プロトコル
  • 追加は GUI (歯車 → コネクター) または .mcp.json で
  • filesystem / github / postgres / puppeteer など定番が多数
  • 権限は mcp__サーバー名__ツール名 単位で許可・拒否できる
