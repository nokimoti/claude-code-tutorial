STEP 07

ファイルを編集させる

このステップのゴール

  • Claude にファイルを書き換えてもらう体験をする
  • 編集前に「許可するかどうか」を聞かれることを理解する

「書き換えて」と頼んでみる

STEP 06 で作った memo.txt をそのまま使います。たとえばこう頼みます。 

memo.txt のリストに「卵 1パック」を追加して

すると、Claude はまず「どこを、どう書き換えたいか」を画面に 差分プレビュー として表示します。 

✏️ Edit(memo.txt)
   - パン 2斤
   + パン 2斤
   + - 卵 1パック

   ┌─────────────────┬──────────────────────┬──────┐
   │  ✓ 許可する     │  ✓ 同じ操作は今後許可 │  ✕ いいえ │
   └─────────────────┴──────────────────────┴──────┘

「許可」を出すまでファイルは変わらない

ここが Claude Code の とても大切なポイント です。 Claude が勝手にファイルを書き換えることはありません。 あなたが「許可する」を押すまで、ファイルはそのまま です。

この仕組みを 許可モード といいます。 初期状態では「ファイル編集」「コマンド実行」など、危険になりうる操作の前に必ず確認ダイアログが出ます。

差分の見かた

確認画面に出てくる差分には意味があります。

  • + 緑色 = 追加される行
  • - 赤色 = 削除される行
  • 変わらない行は、前後の数行が文脈として表示される

アプリによってはサイドバイサイド (左:変更前 / 右:変更後) で表示されることもあります。 差分を見て納得してから「許可する」を押す 習慣をつけましょう。

3 つのボタンの違い

ボタン意味使いどころ
許可する 今回だけ許可 毎回中身を確認したいとき (推奨)
同じ操作は今後許可 このセッション中、同じツールは聞かない 大量に編集が続くと分かっていて、信頼できるとき
いいえ キャンセル 差分を見て「違うな」と思ったとき
慣れないうちは 毎回「許可する」を選ぶ のがおすすめです。 「今後許可」を押した瞬間に大量のファイルを書き換えられて泣く、というのは初心者あるあるです。

新しいファイルを作ってもらう

Edit は既存ファイルの書き換え、Write は新規ファイルの作成や全置換に使われます。 たとえば、こんな依頼で新しいファイルが作れます。 

hello.py という Python ファイルを作って、
「Hello, Claude!」と表示するコードを書いて

Claude が 📝 Write(hello.py) を提案してくるので、内容を確認して許可すれば作成完了です。 作成されたファイルは、すぐにファイラーや好きなエディタで開けます。

気に入らなかったら、戻させればいい

編集後に「やっぱり違った」と思ったら、Claude に「さっきの編集を戻して」と頼めばすぐ戻ります。

Claude Desktop アプリには「直前の状態に戻す」機能 (チェックポイント) もあります。 メッセージにマウスを乗せると現れる「↶」ボタンや、 メニューの「会話を巻き戻す」から、過去のメッセージまで戻れます。

このステップのまとめ

  • 編集を頼むと、Claude はまず差分プレビューを表示して 許可 を求める
  • 「許可する」を押すまでファイルは変わらない
  • 差分を確認してから許可する習慣を持つ
  • 新規作成は Write、書き換えは Edit という ツール を使い分ける
