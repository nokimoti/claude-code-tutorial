STEP 09

許可の仕組み

このステップのゴール

  • 4 つの 許可モード の違いを把握する
  • 自分の用途に合った安全レベルを選べるようになる

許可は「まとめて」設定もできる

STEP 07・08 で、ファイル編集やコマンド実行のたびに「許可する / いいえ」を聞かれることを見ました。 でも、毎回押すのが面倒なときもあります。 Claude Desktop アプリには、4 段階の 許可モード が用意されています。

4 つのモード

モード動き向いている用途
標準 (Default) 編集・コマンド実行のたびにダイアログで確認 慣れないうち、本番環境
計画 (Plan) 調査・読み取りのみ。一切書き換えない 方針を相談したいとき (STEP 12 で詳しく)
編集を自動許可 (Accept Edits) ファイル編集は自動許可、コマンド実行は確認 大量に編集が続くと分かっているとき
すべて許可 (Bypass・危険) すべて確認なし 使い捨ての環境、隔離されたコンテナ内

切り替え方

入力欄の近くにある モード切替ボタン (盾やシールドのアイコンが多い) をクリックすると、 4 つのモードを順番に切り替えられます。 キーボードでは Shift + Tab でも切り替えられます。

画面下部に現在のモードが常時表示されているので、いま自分がどのモードなのかは一目でわかります。 

🛡  モード: 編集を自動許可 (Shift+Tab で切替)

細かい許可ルールはアプリの設定画面 / settings.json で

「npm install はいつも許可」「rm は絶対に許可しない」のように、 コマンド単位で細かい許可ルールを書きたい場合は、設定画面 から行えます。

画面左下の歯車アイコン (⚙) →「権限」を開くと、 「いつも許可するコマンド」「絶対に拒否するコマンド」のリストが編集できます。 上級者は同じ内容を settings.json に直接書くこともできます。 

{
  "permissions": {
    "allow": [
      "Bash(npm test:*)",
      "Bash(git status)",
      "Read(*)"
    ],
    "deny": [
      "Bash(rm -rf:*)"
    ]
  }
}

「すべて許可」モードの注意

「すべて許可」モード (Bypass) は強力ですが、 Claude が予期しないコマンドを実行しても止められません。 本番環境やメインの作業フォルダでは絶対に使わないでください。 使うなら Docker コンテナや使い捨ての VM の中だけに限定しましょう。

慣れるまでの推奨設定

  1. 最初の数日は 標準モード のままで使う
  2. 毎回「許可する」を押して、何が起きているか観察する
  3. 慣れてきたら 編集を自動許可 モードに切り替えてみる
  4. 頻繁に許可するコマンド (git status など) は設定画面から登録する
入力欄で /permissions と打つと、いま許可されている操作と禁止されている操作の一覧がチャット欄に表示されます。

このステップのまとめ

  • 許可モードは 標準 / 計画 / 編集を自動許可 / すべて許可 の 4 段階
  • モード切替ボタンか Shift + Tab で切り替えられる
  • 細かいルールは設定画面 (or settings.json) で書ける
  • 「すべて許可」は必要なときだけ、隔離された環境でだけ使う
