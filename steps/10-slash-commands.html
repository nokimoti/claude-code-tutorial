STEP 10

スラッシュコマンド

このステップのゴール

  • 覚えておくと便利な スラッシュコマンド を 5 つ覚える
  • 自分でカスタムスラッシュコマンドを作れることを知る

「スラッシュコマンド」とは?

入力欄の先頭に / を打つと、Claude にではなく Claude Code 自体 に対する命令を送れます。 これを スラッシュコマンド といいます。 入力中に候補メニューが出るので、矢印キーで選んで Enter で実行できます。

覚えておきたい 5 つ

1. /help — 困ったらまずこれ

使えるコマンド一覧が表示されます。新しい機能が増えていることがあるので、 時々眺めてみると発見があります。

2. /clear — 会話をリセット

これまでの会話履歴をすべて忘れさせて、まっさらな状態にします。 話題が大きく変わったときに使うと、コンテキストウィンドウ を節約できて便利です。

話題が変わるたびに /clear する習慣をつけると、料金もパフォーマンスもよくなります。

3. /init — プロジェクトを Claude に把握させる

プロジェクトのフォルダ構成・使っている言語・テストの動かし方などを Claude が調べて、 CLAUDE.md という覚書ファイルを作ってくれます。 新しいプロジェクトで Claude Code を初めて使うときの最初の一手です (詳しくは STEP 11)。

4. /exit — 終了

セッションを終了します。Ctrl + C を 2 回押すのと同じ効果です。

5. /permissions — 許可状況を確認

いま設定されている許可ルールを一覧で表示します。 「あれ?なんでこのコマンドが自動で通るんだっけ?」と思ったときの確認に。

その他のよく使うもの

  • /model — 使う AI モデルを切り替え (Sonnet ⇄ Opus など)
  • /login / /logout — Claude アカウントの再認証
  • /status — 現在の設定や接続状況を表示
  • /agents — サブエージェント の管理
  • /mcp — MCP サーバーの管理
  • /config — 設定画面を開く
  • /review — 現在の変更をレビュー

自分で作れる「カスタムスラッシュコマンド」

よく使うプロンプトをスラッシュコマンドとして登録できます。 たとえば .claude/commands/test.md という Markdown ファイルを作って、 中身に 

テストを実行して、失敗しているテストがあれば
原因と修正方針を教えてください。

と書いておくと、入力欄で /test と打つだけでこの依頼が送れるようになります。

コマンドファイルの中で $ARGUMENTS という記号を書くと、引数を受け取れます。 /translate 英語 のような使い方もできます。

このステップのまとめ

  • 入力欄で / から始めると スラッシュコマンド
  • まずは /help /clear /init /exit /permissions を覚える
  • よく使うプロンプトはカスタムコマンドにして再利用できる
