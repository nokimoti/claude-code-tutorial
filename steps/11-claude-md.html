STEP 11

CLAUDE.md でプロジェクトを覚えてもらう

このステップのゴール

  • CLAUDE.md の役割を理解する
  • /init で自動生成、または自分で書いて Claude に常時参照させる

毎回同じ説明をするのは疲れる

アプリで新しい会話を始めるたびに「このプロジェクトでは TypeScript を使ってます」「テストは npm test で動きます」 「コミットメッセージは英語で書いてください」…と毎回伝えるのは大変です。

そんなときに使うのが CLAUDE.md という特別なファイルです。

CLAUDE.md とは

プロジェクトのルートに CLAUDE.md という名前のファイルを置いておくと、 Claude Desktop アプリは 会話開始時に必ず読み込んで、その内容を踏まえて応答します。 Markdown 形式で書ける、Claude 専用の指示書です。

自動で作ってもらう

新しいプロジェクトで使うときは、まず入力欄でこう打ちます。 

/init

Claude が自動でプロジェクト構造を分析して、CLAUDE.md の雛形を提案してくれます。 内容を確認して「許可する」を押せば作成完了です。

自分で書く場合のサンプル

手書きする場合は、こんな内容を書いておくと便利です。 Claude に「CLAUDE.md を作って、以下の内容を書いて」と頼んでもいいですし、 普通のテキストエディタ (メモ帳・TextEdit など) で直接書いても OK です。 

# プロジェクト概要
社内向けタスク管理アプリのバックエンド。

# 使っている技術
- 言語: TypeScript (Node.js 20)
- フレームワーク: Express
- DB: PostgreSQL
- テスト: Vitest

# よく使うコマンド
- 開発サーバ起動: `npm run dev`
- テスト実行: `npm test`
- 型チェック: `npm run typecheck`

# コーディングルール
- セミコロンは付ける
- インポートは絶対パスではなく相対パスを使う
- 関数の説明コメントは原則書かない (型と関数名で伝える)

# Claude へのお願い
- ファイル編集前にテストの内容も確認してください
- console.log を残したままにしないでください
- 大きな変更は計画モードで提案してから進めてください

2 種類の CLAUDE.md

  • プロジェクト用 = ./CLAUDE.md (プロジェクトフォルダ直下)
    そのプロジェクトを触る人全員に効きます。Git にコミットします。
  • 個人用 = ~/.claude/CLAUDE.md (ホームディレクトリ)
    自分のパソコンでだけ効く。「日本語で答えて」など個人の好みを書きます。 アプリの設定画面の「グローバル指示」から GUI で編集することもできます。

編集したらすぐ反映される?

CLAUDE.md を編集しても、実行中の会話には反映されません。 /clear で会話をリセットするか、新しい会話を始めると、新しい CLAUDE.md が読み込まれます。

入力欄で # から始めて何か書くと、それが CLAUDE.md に追記されます。 「いま気づいた注意点を残しておく」のに便利です。

長すぎる CLAUDE.md は逆効果

毎回読み込まれる = 毎回 トークン を消費する、ということです。 数千行の長文を書くと、会話のスタート時点で大量のコンテキストを食ってしまいます。

長くなりそうなら、参照用ドキュメントを別ファイルに分けて、CLAUDE.md からは 「詳細は docs/architecture.md を参照」のようにリンクで指すのがおすすめです。

このステップのまとめ

  • CLAUDE.md はプロジェクト固有の指示書
  • /init で自動生成できる
  • プロジェクト用とユーザー個人用の 2 種類がある
  • 長すぎると逆効果。要点だけ書く
