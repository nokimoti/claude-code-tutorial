このステップのゴール

  • ここから先に学べる、もう少し進んだ機能の名前と用途を知る

おつかれさまでした

ここまで読んだあなたは、Claude Code の基本操作を一通り把握しています。 ファイルを読ませる・書き換えさせる・コマンドを実行させる・計画を立てさせる ── これだけで、毎日の作業時間がかなり変わるはずです。

ここから先は、もっと使い込みたくなったときに見る「次のメニュー」を紹介します。 今すぐ理解する必要はありません。こういう機能があるんだな と頭の片隅に置いておいてください。

1. IDE 連携

VS Code や JetBrains 系の IDE に Claude Code 拡張をインストールすると、 エディタ内に Claude Code のサイドパネルが開いて、Claude Desktop アプリと IDE を行き来する手間が減ります。 また、IDE で開いているファイルが自動でコンテキストとして共有されます。

2. Git を組み合わせる

Git でバージョン管理しているプロジェクトでは、Claude Code の真価が出ます。

  • 編集前に commit しておけば、いつでも元に戻せる
  • 「PR 作って」と頼めば gh pr create までやってくれる
  • 「最近のコミットを要約して」のようなレビュー支援も得意

3. サブエージェント

サブエージェント は、特定の役割を持つ「分身」のような仕組みです。 たとえば「コードレビュー専門」「テスト実行専門」のサブエージェントを定義しておくと、 メインの Claude がそのときだけ専門家を呼び出して仕事を分担できます。 入力欄で /agents から管理できます。

4. MCP サーバー

MCP (Model Context Protocol) は、 Claude に「外部システムへの手足」を生やす仕組みです。 MCP サーバーを接続すると、たとえば

  • Notion や Slack の中身を Claude に読み書きさせる
  • 社内のデータベースに直接クエリを投げさせる
  • ブラウザを操作させる

といった連携ができるようになります。 アプリの設定画面の「コネクター」または「MCP」項目から GUI で追加できるものもあります。

5. フック

フック は「ある操作の前後に、決まったコマンドを自動で走らせる」仕組みです。

  • ファイル編集後、自動で linter を走らせる
  • セッション終了時、自動で git status を表示する
  • 特定の操作を完全に禁止する

こうしたルールを settings.json に書いておけます。

6. スキル

Claude が「PDF を読む」「Excel を編集する」など、専門的な作業を行うための小さな知識パックです。 公式が用意しているもののほか、自分で作って共有することもできます。

7. コマンドライン版 の Claude Code

Claude Code には、ターミナル上で動く CLI 版もあります。 開発に慣れた人は、エディタ脇のターミナルで claude コマンドを叩いて使うのが好きな場合もあります。 Desktop アプリに慣れたあと、興味があれば触ってみてください。

公式情報の入り口

  • 公式ドキュメント: docs.claude.com
  • ヘルプ: 入力欄で /help
  • 不具合報告: GitHub の issues
迷ったら、Claude 自身に 「Claude Code のフックの使い方を教えて」 のように聞いてみるのが早いです。 Claude Code に関する説明は Claude が一番よく知っています。

このサイトのまとめ

  • Claude Code は AI コーディングアシスタント。Claude Desktop アプリで使うのがいちばん手軽
  • ファイル読み書き・コマンド実行を「お願いベース」で進められる
  • 編集の前には必ず差分が出て、許可を求められる (安全)
  • 大きな仕事はプランモードから・区切りで /clear・困ったら /help
  • もっと深く使うなら IDE 連携・サブエージェント・MCP・フック・スキル・CLI 版
