STEP 08

コマンドを実行させる

このステップのゴール

  • Claude にコマンドを実行してもらう体験をする
  • 「テストを走らせる」「ビルドする」などの作業を任せられるイメージを持つ

「実行して」と頼むだけ

STEP 07 で hello.py を作ったなら、こう頼んでみましょう。 

hello.py を実行して結果を見せて

Claude は次のような実行内容を提案してきます。 

⚡ Bash(python hello.py)
   このコマンドを実行してもよいですか?

   [ ✓ 実行する ]  [ ✓ 同じ操作は今後許可 ]  [ ✕ キャンセル ]

「実行する」をクリックすると Claude がコマンドを動かして、結果を画面に表示します。 

✓ 実行完了
  Hello, Claude!

Bash という ツール が、コマンド実行のための手足です。 初めて実行する種類のコマンドは、このように毎回確認画面が出ます。

裏ではアプリ内蔵の「シェル」が動く

Claude Desktop アプリには、コマンドを実行するための仕組みが組み込まれています。 自分でターミナルを開かなくても、アプリ内ですべて完結します。 OS によって動くシェルは違います。

  • Windows: PowerShell
  • macOS / Linux: bash または zsh

Claude Code は OS を判別して、適切なシェル文法でコマンドを組み立ててくれます。

こんなお願いができる

  • 「テストを実行して、失敗しているところを教えて」
  • 「npm install を実行してログを見せて」
  • 「このフォルダの中で TODO という単語を含むファイルを探して」
  • 「git status と git log -5 を実行して、何が起きているか教えて」

Claude は結果を読んで 意味を要約 してくれるので、 長いログの中から本質を抜き出すのが得意です。

危険なコマンドには特に強い警告

rm -rf や git push --force、 パッケージのアンインストールなど、取り消しが難しいコマンド には アプリが特別な警告ダイアログを出します。背景が赤くなって目立つようになっています。

警告が出たら必ず立ち止まって読んでください。 「とりあえず許可」を押し続けるとプロジェクトが壊れる可能性があります。

長く動くコマンドはバックグラウンドへ

npm run dev のような「ずっと動き続ける」コマンドは、 Claude が自動でバックグラウンド実行してくれます。 その間も会話は続けられ、必要に応じてログを覗いてくれます。 アプリ画面の上部や下部に「実行中のプロセス」インジケーターが表示されることもあります。

ネット接続が必要なコマンド

npm install や git pull のように外部ネットワークに繋ぎに行くコマンドも普通に実行できます。 ただし、企業ネットワークなどでプロキシ設定が必要な場合は、事前に環境を整えておきましょう。

コマンドの出力が長すぎるときは「結果を 30 行に絞って要約して」のように頼むと、 コンテキストウィンドウ を節約できます。

このステップのまとめ

  • 「実行して」と頼めば Bash ツールが動く
  • 毎回ダイアログで許可を求められる (危険なコマンドは特に強く警告される)
  • テスト実行・ビルド・git 操作など、コマンドラインの作業の多くを任せられる
  • 自分でターミナルを開く必要はなし、アプリ内で完結
