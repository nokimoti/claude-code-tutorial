STEP 06

ファイルを読ませる

このステップのゴール

  • Claude にプロジェクトのファイルを読ませて、内容について質問できる
  • 「ツール」という考え方を理解する

練習用のファイルを置いてみる

STEP 04 で開いた claude-test フォルダの中に、 お使いの OS のファイラー (Windows のエクスプローラー、macOS の Finder) で 適当なテキストファイルを置きましょう。

たとえば、メモ帳 / TextEdit で次のような内容を作って、memo.txt として保存します。 

memo.txt:
今日の買い物リスト
- りんご 3個
- 牛乳 1本
- パン 2斤

2 つの読ませ方

方法 A: そのまま聞く (おすすめ)

アプリの入力欄で、ファイル名を含めて頼んでみます。 

memo.txt の中身を読んで、買い物リストを箇条書きで教えて

Claude は「あ、プロジェクトフォルダにあるファイルだな」と判断して、 自動でそのファイルを開きます。画面にこういう表示が流れるはずです。 

📄 Read(memo.txt)
   ✓ 4 行を読み込みました

はい、読みました。買い物リストは以下のとおりです:
- りんご 3個
- 牛乳 1本
- パン 2斤

方法 B: ドラッグ & ドロップで添付

ファイラーから ファイルを直接アプリの入力欄にドラッグ しても OK です。 入力欄にファイル名のチップが表示され、メッセージと一緒に送れます。 プロジェクトフォルダの外にあるファイルを読ませたいときは、こちらが便利です。

入力欄にある「クリップ (📎)」アイコンをクリックして、ファイル選択ダイアログから添付することもできます。

「ツール」が裏で動いている

画面に出た 📄 Read(memo.txt) という表示に注目してください。 これは「Claude が Read という ツール を使ってファイルを読んだ」という意味です。

Claude 自体は、頭の中で考えて文章を返すだけの存在です。 でもファイルを読んだりコマンドを実行したりする「手足」として、いくつかの ツール が用意されています。

  • Read — ファイルの中身を読む
  • Edit — ファイルの一部を書き換える
  • Write — ファイルを新しく書く / 上書きする
  • Bash — シェルのコマンドを実行する
  • Glob / Grep — ファイル名や中身を検索する

Claude はあなたのお願いを聞いて、必要なツールを自分で選んで使います。

大きすぎるファイルには注意

Claude が一度に読み込める量には限界があります。これを コンテキストウィンドウ と呼びます。 数万行を超える巨大なログファイルなどは、一度に全部読ませると失敗することがあります。

その場合は「最初の 200 行だけ見て」「エラー部分だけ抜粋して読んで」のように、 範囲を絞って依頼するとうまくいきます。

フォルダ全体について聞いてみる

個別のファイルだけでなく、こんな聞き方もできます。 

このプロジェクトのフォルダ構成を教えて

Claude が自動でファイル一覧を見て、概要をまとめてくれます。 新しく開いたプロジェクトを把握するときに便利です。

このステップのまとめ

  • ファイル名を伝えるだけで Claude が読んでくれる
  • 外のファイルはドラッグ & ドロップでも添付できる
  • 裏では Read という ツール が動いている
  • 大きなファイルは コンテキストウィンドウ の制限に注意
