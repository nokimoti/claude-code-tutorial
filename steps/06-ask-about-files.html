STEP 06

ファイルを読ませる

このステップのゴール

  • Claude にローカルのファイルを読ませて、内容について質問できる
  • 「ツール」という考え方を理解する

練習用のファイルを置いてみる

STEP 04 で作った claude-test フォルダの中に、適当なテキストファイルを置きましょう。 以下のような memo.txt を用意してみてください。 

memo.txt:
今日の買い物リスト
- りんご 3個
- 牛乳 1本
- パン 2斤

そのまま聞いてみる

Claude Code を起動した状態で、こんなふうに頼んでみます。 

> memo.txt の中身を読んで、買い物リストを箇条書きで教えて

画面にこういうメッセージが流れるはずです。 

● Read(memo.txt)
  ⎿  Read 4 lines

  はい、読みました。買い物リストは以下のとおりです:
  - りんご 3個
  - 牛乳 1本
  - パン 2斤

「ツール」が裏で動いている

● Read(memo.txt) という表示に注目してください。 これは「Claude が Read という ツール を使ってファイルを読んだ」という意味です。

Claude 自体は、頭の中で考えて文章を返すだけの存在です。 でもファイルを読んだりコマンドを実行したりする「手足」として、いくつかの ツール が用意されています。

  • Read — ファイルの中身を読む
  • Edit — ファイルの一部を書き換える
  • Write — ファイルを新しく書く / 上書きする
  • Bash — シェルのコマンドを実行する
  • Glob / Grep — ファイル名や中身を検索する

Claude はあなたのお願いを聞いて、必要なツールを自分で選んで使います。

ファイル名の渡し方いろいろ

ファイルを読ませる方法はいくつかあります。

  1. ファイル名をそのまま書く — 「memo.txt を見て」
  2. パスを書く — 「./docs/spec.md を要約して」
  3. @ 記号で添付する — 入力中に @ を打つとファイル選択メニューが出ます
  4. 「このファイル」と書く — 直前の文脈にあるファイルを指す

大きすぎるファイルには注意

Claude が一度に読み込める量には限界があります。これを コンテキストウィンドウ と呼びます。 数万行を超える巨大なログファイルなどは、一度に全部読ませると失敗することがあります。

その場合は「最初の 200 行だけ見て」「エラー部分だけ抜粋して読んで」のように、 範囲を絞って依頼するとうまくいきます。

フォルダ全体について聞きたいときは「このプロジェクトの構造を教えて」と聞いてみると、 Claude が自動でファイル一覧を見て、概要をまとめてくれます。

このステップのまとめ

  • ファイル名を伝えるだけで Claude が読んでくれる
  • 裏では Read という ツール が動いている
  • 大きなファイルは コンテキストウィンドウ の制限に注意
